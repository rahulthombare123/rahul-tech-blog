React Hooks: A Deep Dive into useEffect

React January 10, 2026 Rahul

The useEffect hook is essential for handling side effects in functional components. Let's explore its dependencies, cleanup patterns, and common use cases.

What is useEffect?

useEffect lets you perform side effects in function components. It serves the same purpose as componentDidMount, componentDidUpdate, and componentWillUnmount in class components, but unified into a single API.

Basic Usage

import { useEffect } from 'react';

function MyComponent() {
    useEffect(() => {
        // Code here runs after every render
        document.title = 'New Title';
    });
}

Tip: To run an effect only once (on mount), pass an empty dependency array: useEffect(() => {...}, []).

Dependency Array

The second argument controls when the effect re-runs:

  • No array: Runs after every render.
  • Empty array []: Runs only once after initial render.
  • Array with values: Runs only when those values change.
useEffect(() => {
    console.log('Count changed:', count);
}, [count]); // Only re-run if count changes

Cleanup Function

Return a function from useEffect to clean up (e.g., remove event listeners, cancel subscriptions).

useEffect(() => {
    const timer = setInterval(() => {
        console.log('Tick');
    }, 1000);

    return () => clearInterval(timer); // Cleanup on unmount
}, []);

Common Pitfall: Forgetting to specify dependencies can lead to bugs. Always include all values from the component scope that the effect uses.

Conclusion

Mastering useEffect is crucial for building robust React applications. Practice with different dependency patterns to understand when effects run.
