Understanding Closures in JavaScript

JavaScript March 15, 2026 Rahul

Closures are one of the most powerful and often misunderstood concepts in JavaScript. In this post, we'll break down what closures are, how they work, and why you should care about them.

What is a Closure?

A closure is a function that has access to its own scope, the outer function's variables, and global variables – even after the outer function has returned. This is possible because functions in JavaScript form closures around the scope in which they are defined.

Basic Example

function outerFunction(outerVariable) {
    return function innerFunction(innerVariable) {
        console.log('Outer Variable: ' + outerVariable);
        console.log('Inner Variable: ' + innerVariable);
    }
}

const newFunction = outerFunction('outside');
newFunction('inside'); // Logs: Outer Variable: outside, Inner Variable: inside

Tip: Even though outerFunction has finished executing, innerFunction still remembers the value of outerVariable. That's the closure in action!

Why Use Closures?

Closures are useful for data privacy, creating function factories, and maintaining state in asynchronous code. Here are a few practical use cases:

  • Private Variables: You can emulate private methods by using closures.
  • Event Handlers: Closures preserve state when attaching event listeners.
  • Partial Application: Pre‑set function arguments with closures.

Example: Private Counter

function createCounter() {
    let count = 0;
    return {
        increment: function() { count++; },
        decrement: function() { count--; },
        getCount: function() { return count; }
    };
}

const counter = createCounter();
counter.increment();
counter.increment();
console.log(counter.getCount()); // 2

Common Pitfall: Closures can cause memory leaks if you're not careful, because they keep references to outer variables. Always nullify references when they're no longer needed.

Conclusion

Closures are a fundamental concept that every JavaScript developer must understand. They enable powerful patterns and are everywhere in modern JavaScript – from React hooks to event listeners. Practice writing your own closures to get comfortable with them.
